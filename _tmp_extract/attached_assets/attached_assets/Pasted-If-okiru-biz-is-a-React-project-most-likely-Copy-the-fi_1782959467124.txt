If okiru.biz is a React project (most likely)
Copy the file OkiruForm.tsx into the okiru.biz Replit project as-is — it has no dependencies beyond React, and its default apiBase (https://okiru.app/api) already points at this CRM's live API.
Drop it in wherever the current Zoho Forms embeds are, one per form:

<OkiruForm formId="okiru-biz-contact" />
<OkiruForm formId="okiru-biz-training" />
<OkiruForm formId="okiru-biz-mailing" />
Remove the old Zoho Forms embeds for these three forms so leads aren't split across two systems.
No secrets/API keys needed — /api/leads/intake and /api/forms/* are already open to all origins (checked in server.ts's PUBLIC_PATHS), so nothing to configure on Replit's env vars for this.
Discard okiru-leads-endpoint.mjs entirely — it targeted a different, never-built Railway service and is now superseded by this direct path.
If okiru.biz isn't React (plain HTML/JS, another framework)
Replicate what OkiruForm.tsx does, in two calls:


// 1. Fetch field definitions to render the form
const config = await fetch("https://okiru.app/api/forms/okiru-biz-contact/config").then(r => r.json());

// 2. On submit
await fetch("https://okiru.app/api/leads/intake", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    formId: "okiru-biz-contact",   // or okiru-biz-training / okiru-biz-mailing
    fields: { name, email, phone, company, topic, message },  // match keys from config.fields
    consent: true,
    attribution: { utmSource, utmMedium, utmCampaign, referrer, landingUrl: location.href },
  }),
});
